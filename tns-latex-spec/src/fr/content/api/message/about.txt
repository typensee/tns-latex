===============================
Localiser le "lanceur d'alerte"
===============================

Chaque message indique toujours les \infos suivantes.

    1) **Le nom du \pack.** Pour les \packs "agrégateurs", les erreurs qui ne leur sont pas propres garderont la \ref au sous-\pack utilisé afin de vite cibler un \pb et rendre plus efficace la correction d'eventuels bogues.

    1) **Le contexte.** Ceci peut concerner l'activation d'un module, l'emploi d'une macro ou d'un \env.

    1) **Le pourquoi.** Une \desc courte, mais précise, permet d'aider à la fois l'utilisateur et le maintenur du code en cas de bogue.
