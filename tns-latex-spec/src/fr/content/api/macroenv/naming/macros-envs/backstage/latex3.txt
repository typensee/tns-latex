============
Avec \latex3
============

Les macros et les \envs sont codés en coulisse en \latex3 a minima : ceci signifie que chaque package propose une \api publique faisant le lien avec les "fonctions" \latex3 qui font réellement le travail en coulisse.
Pour ces macros et ces \envs privés, les conventions de nommage du projet \latex3 sont respectées
((
    Le document "The expl3 package and LATEX3 programming (released 2023-03-30)", indique que le nom d'une fonction doit être du type ``\<module>_<description>:<arg-spec>`` et celui d'une variable de la forme ``\<scope>_<module>_<description>_<type>`` où ``<scope>`` est au choix ``l`` pour "local", ``g`` pour "global" ou ``c`` pour "constante".

    De plus, ``\module_foo:nnn`` sera une fonction publique, tandis que ``\__module_foo:nnn`` sera une fonction privée.
    Ce principe s'applique comme suit aux variables : ``\l_module_foo_tl`` est publique, tandis que ``\l__module_foo_tl`` est privée.
)).
Côté casse de la police, on reprend ce qui a été indiqué pour les outils "utilisateur".
