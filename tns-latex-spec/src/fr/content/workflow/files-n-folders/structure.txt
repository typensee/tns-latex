==============================
Structure minimale d'un projet
==============================

Considérons un projet fictif nommé ``tnsfictive``. Son \dev se fera dans un dossier path::``tnsfictive`` ayant le contenu minimal suivant.

dirtree::
    + tnsfictive
        * about.yaml
        + changes  [...]
        + src      [...]


Voici une présentation rapide de ces composants, en attendant des compléments dans des sections à venir.

    1) Le nom du dossier principal correspond au nom du package.

    1) Le fichier path::``about.yaml`` contient des \metainfos à propos du projet ; voir cf::``?/metainfos.txt``.

    1) Le dossier path::``changes`` permet d'indiquer les \chgts comme expliqué dans cf::``?/workflow/changes.txt``.

    1) Le dossier path::``src`` est le code \src du projet à développer en suivant les règles indiquées dans cf::``?/coding/small-code.txt``.


On peut utiliser en plus, mais de façon optionnelle, les dossiers suivants.

    1) Un dossier path::``contribute`` permet d'accueillir des contributions extérieures qui sont majoritairement des \trads (dans ce cas, la procédure devra être accessible à des non codeurs) ; voir cf::``?/translate.txt`` à ce propos.

    1) Un dossier path::``readme`` permet de rédiger par petits bouts un fichier \readme de présnetation du projet ; des explications sont données dans cf::``readme.txt``.
