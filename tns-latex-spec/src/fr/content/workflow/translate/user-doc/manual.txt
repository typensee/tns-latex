=======================
Le manuel d'utilisation
=======================

La \doc "utilisateur" rédigée par le codeur sert de point de départ aux \trads. Imaginons que la langue du codeur soit le français
((
    Ceci s'indique via le fichier path::``about.yaml`` du projet.
)).
Dans ce cas, le dossier path::``contribute/doc/manual/fr``, comme ci-dessous, contiendra la dernière version
((
    Le processus de \maj est automatisé.
))
de cette \doc rédigée morceau par morceau et non globalement pour des raisons pratiques.

dirtree::
    + tnsfictive
        + contribute
            + doc
                + manual
                    + fr [...]
                + [...]
            + [...]
        + [...]


Chacun des fichiers de path::``contribute/doc/manual/fr`` est directement compilable, une fois le \pack visible par \latex. Ceci permet de partager simplement le dossier pour une \nelle \trad.
Concrètement, un traducteur espagnol copiera et collera le contenu de path::``contribute/doc/manual/fr`` dans un dossier de son choix, mais nommé path::``es`` où ``es`` vient du codage de la \isolangnorme.
Une fois le travail fini, deux méthodes sont possibles pour proposer la \nelle \trad.

    1) **Le traducteur ne connaît pas \github.**
       Dans ce cas, les étapes à suivre sont les suivantes.

        a) Compression au format \zip du dossier de la \trad.

        a) Envoi du dossier compressé à l'adresse email::``projetmbc@gmail.com`` en indiquant bien "tnsfictive - Contribute - Doc" comme sujet du courriel, avec bien entendu le bon nom du \pack au lieu de ``tnsfictive``.

    1) **Le traducteur connaît \github.**
       Dans ce cas, les étapes à suivre sont les suivantes.

        a) Clonage du projet.

        a) Ajout de la \trad dans le dossier path::``contribute/doc/manual``.

        a) Utilisation d'un git::``git merge requests``.
