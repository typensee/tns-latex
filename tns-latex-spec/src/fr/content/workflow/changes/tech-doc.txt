================
Pour les codeurs
================

Les \chgts de la \doc en anglais s'indiquent dans des fichiers de type ext::``txt`` organisés comme dans l'exemple fictif suivant où tout se passe dans le dossier path::``changes`` en utilisant pour chaque année un dossier bien nommé et pour chaque mois un fichier bien nommé lui aussi.

dirtree::
    + tnsfictive
        + changes
            + 2020
                * 06.txt
            + 2021
                * 03.txt
                * 07.txt
        + [...]


Voici un contenu possible pour le fichier path::``06.txt``
((
    Cette syntaxe très intuitive est celle du projet \tnschges.
)).

tns::
    ==
    01  (0.0.0-beta)
    ==

    -----
    :new:
    -----

    [[:api:]]
    This package provides a lot of new fictive features.


La date est indiquée via le titre tns::``01`` entouré par des tnschges::``==``.
Quant au \chgt de version, il s'indique simplement en ajoutant tnschges::``(0.0.0-beta)`` après la date.
Pour le reste, cela permet d'indiquer une nouveauté au niveau de l'\api.
