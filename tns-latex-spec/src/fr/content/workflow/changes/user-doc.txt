=====================
Pour les utilisateurs
=====================

L'organisation des \chgts de la \doc "utilisateur" se fait comme suit en notant que le sous-dossier path::``changelog`` se trouve dans le dossier path::``src`` utilisé pour le \dev du code "morceau par morceau" et non dans celui de l'\ens du projet.
La langue des \chgts est celle de la \doc du développeur, soit pour nous le français.
Noter les noms utilisés pour les dossiers "année" et "mois" ainsi que ceux pour les fichiers "jour" où les deux fichiers path::``14-a.tex`` et path::``14-b.tex`` correspondent à deux versions différentes publiées le même jour
((
    Il arrive quelques fois qu'une version soit publiée un peu hâtivement à tors, typiquement avec une \doc ou un journal des \chgts non mis à jour.
)).

dirtree::
    + tnsfictive
        + changes [...]
        + src
            + changelog
                + 2021
                    + 07
                        * 05.tex
                        * 14-a.tex
                        * 14-b.tex
                        * 15.tex
            + [...]
        + [...]


warning::
    Faire attention aux points suivants.

        1) Contrairement à la \doc \tech, les \modifs se tapent jour par jour. Ceci permet notamment de tester la bonne rédaction de l'écriture des \chgts pour une date donnée.

        1) Si plusieurs versions sont publiées le même jour, il faudra suffixer le numéro du jour avec quelque chose du type path::``-a``, path::``-b``, path::``-c``... \etc.


Voici un exemple possible de \chgts ; ceci va nous donner l'occasion d'expliquer quelques points importants.

latex::
    \begin{bdoctopic}[Un court changement]
        ajout de \bdocmacro{unemacro}.
    \end{bdoctopic}

    \begin{bdoctopic}<Plusieurs changements>
        \begin{itemize}
            \item Modification de \bdocmacro{uneautremacro}.
            \item Création de \bdocenv{unenv}.
        \end{itemize}
    \end{bdoctopic}


Voici ce qu'il faut absolument savoir.

    1) Le \numver n'est pas indiqué puisque cette \info est connue via les \chgts \techs.

    1) Un \chgt court se tape directement via ``\begin{bdoctopic}[...] ... \end{bdoctopic}`` qui prend en argument un titre court entre des crochets et dont le contenu détaille les nouveautés.

    1) Pour des \chgts plus longs, il suffit d'utiliser ``\begin{bdoctopic}<...> ... \end{bdoctopic}`` en plaçant le titre entre des chevrons et non des crochets.
    Ici un retour à la ligne est fait \auto^t.

    1) Les macros ``\bdocmacro`` et ``\bdocenv`` permettent d'indiquer \resp des macros et des \envs.

    Un fichier path::``test_one_date.tex``
    ((
        Ce fichier ne fera pas partie du produit final.
    )),
    directement compilable sert à tester et déboguer facilement le rendu des \chgts pour une date donnée.
