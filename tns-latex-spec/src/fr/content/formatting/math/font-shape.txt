==============================
Styles de police de caractères
==============================

La \mef par défaut, hors de tout choix spécifique d'une police de caractères, suit les règles suivantes.

    1) **Forme droite via ``\mathrm`` :** ceci concerne les objets standards, sauf les \enss.

        * Les \foncs réelles ou complexes classiques comme `ln` ou `exp`.

        * Les \ctes réelles ou complexes classiques telles que `pi`, `e` ou `i`.

        * Les opérateurs classiques comme une probabilité `proba(P)`, ou aussi le noyau d'une \appli linéaire `ker`.

    1) **Italique via ``\mathit`` :* ceci concerne les objets modifiables suivant le contexte, sauf les \enss.

        * Les \vars dans une formule.

        * Les noms de \pts, de vecteurs, de suites, de \vars \aleas... \etc.

    1) **Calligraphié via ``\mathcal`` :* ceci concerne des \enss particuliers.

        * Les \enss géométriques telles une droite ou un cercle.

        * Les espaces d'intégration comme `set(L, function, integrable, 2)` l'\ens des \foncs de carré intégrable.

    1) **"Doublement barré" via ``\mathbb`` :* ceci concerne des \enss classiques.

        * Les \enss classiques tels que `RR` et `CC`.

        * Les \enss avec une structure algébrique simple tel un groupe `set(G, grp)`, ou un anneau `set(A, ring)`.

    1) **"À la fraktur" via ``\mathfrak`` :* ceci concerne juste pour le moment les algèbres de Lie comme `set(g, algebra, lie)`.


note::
    Les macros ``\mathbf``, pour le gras, ``\mathsf``, pour l'absence d'empattement et ``\mathtt``, pour le style "machine à écrire", ne font pas partie de l'\api par défaut même si rien n'empêche de les utiliser ponctuellement : par exemple, il est possible d'employer des vecteurs en gras sans flèche à la sauce anglaise.
