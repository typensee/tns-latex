====================
Couleurs... ou pas !
====================

Par défaut, les fonctionnalités proposées sont toutes faites pour un mode noir-et-blanc
((
    Pensons à l'impression de ¨docus.
)).
Il est tout de même possible d'obtenir facilement un rendu coloré
((
    Ceci est utile pour une lecture sur écran.
)).
Ceci se fait via une option de package demandant d'utiliser l'un des thèmes proposés, le noir-et-blanc étant celui par défaut.


warning::
    Les versions colorées et celle noir-et-blanc peuvent différées dans leur ¨mef : on ne fera pas juste un passage à un niveau de gris car ceci est rarement pertinent pour des ¨docus destinés à être photocopiés.


info::
    En coulisse, des macros sont utilisées pour styliser les outils proposés par ¨tnstex ; celles-ci sont fournies par le package ¨tnscoltex qui est l'un des ¨elts du projet ¨tnscol
    ((
        Le but du projet ¨tnscol est de fournir des thèmes utilisables dans différents contextes dont ¨latex, ¨python et ¨html.
        Les thèmes sont "sémantiques" : par exemple, on propose des outils pour mettre en forme un avertissement, et non une couleur nommée "rouge".
        Ce ¨syst s'appuie sur la couleur, mais aussi sur des ¨decos comme l'emploi de cadre, l'usage de pointillés... ¨etc.
    )).
    Se reporter à ¨tnscoltex afin de pouvoir employer les styles dans vos propres créations.
