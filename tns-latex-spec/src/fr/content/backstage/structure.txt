==============================
Structure minimale d'un projet
==============================

Voici ce que doit contenir a minima le dossier d'un projet qui pour nous se nommera ``tnsfictif``.

path-tree::
    + tnsfictif
        * about.peuf
        + changes    [...]
        + doc        [...]
        + readme     [...]
        + src        [...]


Dans les grandes lignes, chacun de ces composants a l'utilité donnée ci-après.

    1) Le nom du dossier principal correspond au nom du package.

    1) Le fichier path::``about.peuf`` contient des méta-informations à propos du projet. Voir cf::``?../metainfo/allpack.txt``.

    1) Le dossier path::``changes`` permet d'indiquer les ¨chgts comme expliqué dans cf::``:?../backstage/changes.txt``.

    1) Le dossier path::``doc`` contient la ¨docu "utilisateur" éventuellement traduite. Voir cf::``:?../user-doc/trans.txt`` à ce propos.

    1) Le dossier path::``readme`` sert à rédiger un fichier ¨readme pour les codeurs. Des explications sont données dans cf::``readme.txt``.

    1) Le dossier path::``src`` est le code source du projet à développer en suivant des règles indiquées dans cf::``?../coding/small-code.txt``.


info::
    Le projet ¨tnsfacto permet d'initier un nouveau projet en construisant une structure minimale "clés en main". Il sert aussi à publier un projet.
