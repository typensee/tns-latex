===================
Langues de la ¨doc
===================

Une ¨1ere ¨doc est rédigée en même temps que le code dans la langue du codeur indiquée dans le fichier path::``about.peuf`` du projet.
De façon automatisée, cette ¨doc est mise dans le dossier path::``doc`` du projet, ce répertoire ayant une structure ressemblante à la suivante si l'on a indiqué peuf::``doc = fr`` pour la langue.
Des explications sont données juste après.

path-tree::
    + tnsfictif
        * about.peuf
        + doc
            + changes [...]
            + fr
                + intro
                    * intro.tex
                + alpha-et-omega3
                    * alpha.tex
                    * omega3.tex
                + poor-lonesome-cowboy
                    * lucky-1.tex
                    * lucky-2.tex
                    * rantanplan.tex
        + [...]


Voici ce qu'il faut savoir pour traduire la ¨doc non ¨tech.

    1) Le contenu du dossier path::``doc/fr`` vient de ce que le codeur a indiqué comme faisant partie de la ¨doc de type tutoriel.

    1) Une ¨nelle ¨trad se commence juste en ajoutant un dossier ayant le même contenu que le dossier path::``doc/fr``.
    Le traducteur aura ensuite à traduire ce contenu directement compilable fichier par fichier via la syntaxe ¨latex, mais sans besoin de mettre les mains dans le cambouis.
    Le nom utilisé pour le dossier utilise le codage des langues suivant la ¨isolang. Par exemple, pour l'anglais et l'espagnol, on utilisera ``en`` et ``es`` ¨resp.

    1) Le dossier path::``doc/changes`` sert à indiquer aux traducteurs les ¨chgts liés à toutes les traductions comme par exemple l'ajout d'une ¨nelle section, ou bien la suppression d'une autre.
    Par contre, les ¨chgts mineurs du type corrections orthographiques et grammaticales propres à une langue ne sont pas indiqués (car ils ne concernent pas les autres langues).
    Ce dossier suit les règles ¨techs indiquées dans cf::``?../changes/technical-doc.txt``.
