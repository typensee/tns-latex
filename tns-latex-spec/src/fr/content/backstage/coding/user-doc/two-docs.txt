==========
Deux ¨docs
==========

Chaque projet propose deux ¨docs : l'une technique
((
    Voir cf::``:?../comment/tech-doc.txt`` où est expliqué comment produire cette ¨doc technique.
))
qui s'adresse juste aux codeurs, et l'autre de type tutoriel destinée aux utilisateurs. Cette dernière se tape via ¨latex
((
    Ceci permet de faire des tests en situation réelle : voir cf::``:?../test.txt``.
))
en respectant des petites régles que nous allons expliquer dans les sections suivantes.


info::
    La ¨doc de type tutoriel s'appuie sur les projets ¨bdoc et ¨macroenvsign (se reporter aux ¨docs de ces deux packages).
