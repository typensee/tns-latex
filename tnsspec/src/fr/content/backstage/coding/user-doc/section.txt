================================
Sections - Comment s'y prendre ?
================================

Pour un package autonome la structure sera celle de la ¨doc sans ¨chgt de niveau de profondeur.
Par contre les packages "agrégateurs" relèvent de un le niveau de profondeur.


warning::
    Il faut veiller à n'utiliser que des latex::``\section`` et latex::``\subsection`` pour éviter de mauvaises surprises dans des packages "agrégateurs".


Un titre de section peut être rédigé sur deux lignes maximum. Ceci peut être utile comme le montre l'exemple suivant où la macro ``\texorpdfstring`` permet d'indiquer un titre spécifiquement pour la ¨tdm d'un ¨pdf.

latex::
    \section{\texorpdfstring{Les opérateurs $\partial$ et $d$}%
                            {Les opérateurs "d rond" et "d droit"}}


warning::
    Le repérage des sections est très basique : il interdit d'utiliser des macros comme ``\section`` dans un environnement de type listing ou verbatim.
    Si besoin, il faudra taper ce type de contenu très spécifique dans un fichier isolé pour passer via des macros d'importation de contenu externe.


Pour les signatures des macros et des environnements, les sections doivent respecter les règles suivantes.

    1) Placer dans un même fichier la ¨doc humaine et celle technique relative.
    
    1) Utiliser le même niveau de profondeur que la section documentant les macros et les environnements.

    1) Utiliser le titre "magique" latex::``:techsign:``.

    1) Penser à refléter la structure souhaitée : voir ``\section{:techsign:}`` dans l'exemple suivant.


Voici une structure possible respectant les contraintes précédentes.

latex::
    \section{Vecteurs}

    \subsection{Norme}

    ... % Pas de section "supérieure" à une \subsection ici !

    \section{:techsign:} % Pour reprendre le titre "Vecteurs"

    \subsection{:techsign:} % Signatures des macros et environnements
                            % en lien avec les vecteurs.

    ... % Pas de section "supérieure" à une \subsection ici !


Lors de la production de la ¨doc au complet, une section dédiée "Toutes les fiches techniques"
((
    Pour la version française bien entendu...
))
réutilisera les bons titres.
Concrètement, pour l'exemple précédent il y aura une section nommée "Vecteurs" contenant une sous-section de titre "Norme" avec toutes les fiches techniques.


warning::
    Le codage du traitement des fiches techniques est fragile mais c'est assumé.
